Sección 01

¿Qué es una función?

Una función es un bloque de código reutilizable que recibe entradas, realiza operaciones y devuelve un resultado. Es la unidad fundamental de organización en R.

Idea central

En R, todo lo que ocurre es una llamada a una función — incluso +, [ o if. Entender las funciones es entender el lenguaje.

¿Para qué sirven?

♻️ Reutilización

Escribe la lógica una vez, úsala cien veces. Sin copiar y pegar código.

📦 Encapsulación

Las variables internas no "escapan". Cada llamada tiene su propio espacio limpio.

🧪 Testabilidad

Una función con entradas y salidas claras es fácil de probar y depurar.

📖 Legibilidad

Un buen nombre de función hace el código auto-explicativo.

Tu primera función

El ejemplo más sencillo posible. Cada bloque tiene rm(list=ls()) al inicio para empezar con el workspace completamente limpio.

ejemplo_01.REjecutable
rm(list = ls())  # limpia el workspace

# Definir la función
saludar <- function(nombre) {
  paste("Hola,", nombre)
}

# Llamar a la función
saludar("Ana")
saludar("Luis")
salida
Convención de nombres

En R se usan nombres en snake_case para funciones: calcular_media, limpiar_datos. Usa verbos que describan la acción.

Funciones ya existentes vs. propias

R tiene miles de funciones predefinidas (mean(), sum(), paste()…). Las funciones que defines tú funcionan exactamente igual.

ejemplo_02.REjecutable
rm(list = ls())

# Función propia que usa funciones de R
resumir <- function(x) {
  cat("Media:   ", mean(x),   "\n")
  cat("Mínimo:  ", min(x),    "\n")
  cat("Máximo:  ", max(x),    "\n")
  cat("Desviac: ", sd(x),     "\n")
}

notas <- c(7.5, 8, 6, 9, 5.5, 8.5)
resumir(notas)
salida
Sección 02

Sintaxis y anatomía

Toda función en R tiene la misma estructura. Conocerla te permite leer cualquier código con soltura.

Las partes de una función

anatomía nombre <- function(arg1, arg2 = valor_por_defecto) { # cuerpo: las instrucciones resultado <- arg1 + arg2 resultado # última expresión = valor devuelto }

Función de una línea (sin llaves)

Si el cuerpo es una sola expresión, puedes omitir las llaves. Muy común en funciones cortas.

sintaxis_01.REjecutable
rm(list = ls())

# Con llaves (forma general)
cuadrado <- function(x) {
  x ^ 2
}

# Sin llaves (una sola expresión)
cubo <- function(x) x ^ 3

# Sintaxis moderna con \() (R 4.1+)
doble <- \(x) x * 2

cuadrado(4)
cubo(3)
doble(7)
salida

Funciones anónimas

Una función sin nombre, usada en el momento. Muy habitual en combinación con sapply, lapply o la pipe |>.

sintaxis_02.REjecutable
rm(list = ls())

numeros <- c(1, 4, 9, 16, 25)

# Función anónima con function()
sapply(numeros, function(x) sqrt(x))

# Función anónima con \() (más corta)
sapply(numeros, \(x) sqrt(x))

# Con la pipe nativa |>
numeros |> sapply(\(x) x + 100)
salida
Cuándo usar función anónima

Úsala cuando la lógica es tan simple que nombrarla no añade claridad. Si la usas más de una vez, dale nombre.

Sección 03

Argumentos

Los argumentos son las entradas de una función. R ofrece gran flexibilidad: argumentos por posición, por nombre, con valores por defecto, y un número variable de ellos.

Argumentos posicionales vs. nombrados

args_01.REjecutable
rm(list = ls())

potencia <- function(base, exponente) {
  base ^ exponente
}

# Por posición: base=2, exponente=3
potencia(2, 3)

# Por nombre: el orden no importa
potencia(exponente = 3, base = 2)

# Mezcla: primero posicional, luego nombrado
potencia(2, exponente = 10)
salida

Valores por defecto

Un argumento con valor por defecto es opcional: si no se proporciona, se usa el valor predefinido.

args_02.REjecutable
rm(list = ls())

calcular_iva <- function(precio, tasa = 0.21) {
  precio * tasa
}

calcular_iva(100)          # usa tasa = 0.21 por defecto
calcular_iva(100, 0.10)    # sobreescribe con tasa = 0.10
calcular_iva(200)          # 200 * 0.21
salida

El argumento especial ... (dot-dot-dot)

... permite pasar un número variable de argumentos, o reenviarlos a otra función sin nombrarlos todos.

args_03.REjecutable
rm(list = ls())

# Número variable de argumentos
mi_suma <- function(...) {
  args <- c(...)
  sum(args)
}

mi_suma(1, 2, 3)
mi_suma(10, 20, 30, 40)

# Reenviar ... a otra función (muy común)
mi_cat <- function(...) {
  cat("→ ", ...)  # pasa todo a cat()
}

mi_cat("hola", "mundo", "\n")
salida
Cuidado con ...

Los argumentos en ... no tienen nombre fijo, lo que dificulta la detección de errores tipográficos. Úsalo cuando realmente necesites flexibilidad; para casos comunes, es mejor nombrar los argumentos explícitamente.

Comprueba lo que sabes

¿Qué devuelve potencia(exponente = 2, base = 3) si la función es function(base, exponente) base ^ exponente?
A 8  (porque 2³)
B 9  (porque 3² — los nombres anulan el orden)
C Error — no se pueden mezclar nombres y posición
Sección 04

Valor de retorno

Una función siempre devuelve algo. Entender exactamente qué devuelve — y cómo — es esencial para escribir código predecible.

Retorno implícito

En R, una función devuelve automáticamente el valor de su última expresión evaluada. No hace falta escribir return().

retorno_01.REjecutable
rm(list = ls())

maximo <- function(a, b) {
  if (a > b) a else b   # última expresión = retorno
}

maximo(5, 12)
maximo(8, 3)

# Guardar el resultado
resultado <- maximo(7, 2)
cat("El máximo es:", resultado, "\n")
salida

Retorno explícito con return()

return() interrumpe la función inmediatamente y devuelve el valor indicado. Útil para salidas anticipadas (guardianes).

retorno_02.REjecutable
rm(list = ls())

dividir <- function(a, b) {
  if (b == 0) {
    warning("División por cero")
    return(NA)   # sale aquí si b == 0
  }
  a / b          # retorno implícito en caso normal
}

dividir(10, 2)
dividir(10, 0)
salida

Devolver múltiples valores con una lista

R solo puede devolver un objeto, pero ese objeto puede ser una list con todo lo que necesites.

retorno_03.REjecutable
rm(list = ls())

estadisticas <- function(x) {
  list(
    media   = mean(x),
    mediana = median(x),
    sd      = sd(x),
    n       = length(x)
  )
}

res <- estadisticas(c(4, 7, 2, 9, 5))
res$media
res$sd
res$n
salida
Retorno invisible

Usar invisible(x) devuelve x pero no lo imprime automáticamente en la consola. Lo usan funciones como ggplot() que devuelven un objeto sin querer inundar la consola. Puedes asignarlo igual: p <- ggplot(...).

Una función tiene este cuerpo: x <- 10; y <- 20; x + y. ¿Qué devuelve?
A NULL — no tiene return()
B 30 — la última expresión evaluada
C 20 — el valor de la última asignación
Sección 05

Funciones como objetos

En R las funciones son objetos de primera clase. Puedes guardarlas en variables, pasarlas como argumentos, devolverlas desde otras funciones y meterlas en listas.

Pasar funciones como argumentos

hof_01.REjecutable
rm(list = ls())

# Una función que recibe otra función como argumento
aplicar_a_vector <- function(v, f) {
  sapply(v, f)
}

numeros <- c(1, 4, 9, 16)

aplicar_a_vector(numeros, sqrt)
aplicar_a_vector(numeros, log)
aplicar_a_vector(numeros, \(x) x * 2 + 1)
salida

Devolver funciones (fábricas)

Una función puede devolver otra función. Esto se llama fábrica de funciones y es la base de los closures (tema de la siguiente página).

hof_02.REjecutable
rm(list = ls())

# Fábrica: genera una función multiplicadora
crear_multiplicador <- function(factor) {
  function(x) x * factor
}

doble  <- crear_multiplicador(2)
triple <- crear_multiplicador(3)

doble(5)    # 10
triple(5)   # 15
doble(triple(4))  # 24: composición
salida

Funciones en listas

hof_03.REjecutable
rm(list = ls())

# Lista de funciones: aplicar cada una al mismo dato
transformaciones <- list(
  raiz    = sqrt,
  log     = log,
  inverso = \(x) 1 / x
)

x <- 100
lapply(transformaciones, \(f) f(x))
salida
Funciones de orden superior

Las funciones que reciben o devuelven otras funciones se llaman funciones de orden superior. sapply, lapply, Map, Filter y Reduce son los ejemplos más usados en R base.

Sección 06

Errores frecuentes

Los mismos errores aparecen una y otra vez al aprender funciones. Reconocerlos en la consola ahorra mucho tiempo.

Error 1 — olvidar el paréntesis al llamar

error_01.REjecutable
rm(list = ls())

saludar <- function() cat("Hola!\n")

# ❌ Sin paréntesis: muestra el código de la función, no la ejecuta
saludar

# ✅ Con paréntesis: ejecuta la función
saludar()
salida

Error 2 — la función devuelve NULL sin querer

error_02.REjecutable
rm(list = ls())

# ❌ La última expresión es una asignación → devuelve NULL
calcular <- function(x) {
  resultado <- x * 2   # asignación: invisible, devuelve NULL
}
calcular(5)   # imprime NULL

# ✅ Hay que exponer el valor final
calcular2 <- function(x) {
  resultado <- x * 2
  resultado          # esta línea devuelve el valor
}
calcular2(5)
salida

Error 3 — confundir cat() con devolver un valor

error_03.REjecutable
rm(list = ls())

# ❌ cat() imprime pero devuelve NULL
doble_malo <- function(x) cat(x * 2)

res <- doble_malo(5)
cat("\nres vale:", res, "\n")   # res es NULL, no 10!

# ✅ Devolver el valor; imprimir es tarea de quien llama
doble_bien <- function(x) x * 2

res2 <- doble_bien(5)
cat("res2 vale:", res2, "\n")
salida
Regla de oro

Una función debe devolver el resultado con la última expresión (o return()). Imprimir con cat() o print() dentro es solo para debug o funciones cuyo propósito es mostrar texto. No mezcles los dos roles.

Ejercicio 2 — Normalizar vector

Escribe normalizar(v) que escale un vector numérico al rango [0, 1]. La fórmula es (x - min) / (max - min). Comprueba que el resultado tiene mínimo 0 y máximo 1.

ej02.REditable

  # tu código aquí

salida
▸ Ver solución
ej02_sol.RSolución
rm(list = ls())

normalizar <- function(v) {
  (v - min(v)) / (max(v) - min(v))
}

datos <- c(10, 20, 30, 40, 50)
normalizar(datos)
