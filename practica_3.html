Simulando células madre en médula ósea

Aplicamos funciones, bucles y alcance de variables para construir un modelo de dinámica celular. Por el camino, el código se rompe — y eso es exactamente lo que queremos estudiar.

Coordinación docente

Esta práctica integra conceptos que estás trabajando en Biología Celular y Tisular:

🔬 Células madre — Tema 12 🔄 Ciclo celular — Tema 9 💀 Apoptosis — Tema 10

¿Qué vamos a construir?

Un programa en R que simula cómo evoluciona una población de células madre en la médula ósea a lo largo de varias generaciones. Cada célula madre puede hacer tres cosas:

💀
Apoptosis
prob = 0.1
La célula muere. No genera descendencia.
🔵🔵
División simétrica
prob = 0.3
Genera dos células madre idénticas.
🔵🔴
División asimétrica
prob = 0.6
Una célula madre + una célula sanguínea.

Las células sanguíneas también mueren con probabilidad 0.2 por generación. Solo las células madre se dividen.

La pregunta central

Cuando metemos la lógica en una función y la llamamos varias veces, los contadores no cambian. ¿Por qué? ¿Dónde tienen que vivir las variables para que el programa recuerde el estado entre generaciones?

Paso 1

El modelo biológico

Antes de escribir código, traducimos el problema biológico a pseudocódigo.

¿Qué necesitamos contar?

En cada generación necesitamos saber cuántas células de cada tipo tenemos. Eso son dos contadores que deben persistir entre generaciones:

pseudocodigo.RPseudocódigo
# Estado inicial
madres     <- 100   # células madre
sanguineas <- 0     # células sanguíneas

# Cada generación:
#   Para cada célula madre:
#     sorteo → apoptosis / simétrica / asimétrica
#     si apoptosis  → madres - 1
#     si simétrica  → madres + 1
#     si asimétrica → sanguineas + 1
#
#   Para cada célula sanguínea:
#     sorteo → muere / vive (prob 0.2 / 0.8)
#     si muere → sanguineas - 1

La herramienta: sample()

Para el azar usamos sample(). Ejecuta estas dos líneas y observa que cada ejecución puede dar un resultado diferente:

paso01_sample.REjecutable
# Destino de UNA célula madre
sample(c("apoptosis", "simetrica", "asimetrica"),
       size = 1, prob = c(0.1, 0.3, 0.6))

# Destino de UNA célula sanguínea
sample(c("muere", "vive"),
       size = 1, prob = c(0.2, 0.8))
salida

Una generación completa

Ahora aplicamos sample() a todas las células en un bucle:

paso02_una_gen.REjecutable
madres     <- 100
sanguineas <- 0

for (i in 1:madres) {
  destino <- sample(c("apoptosis", "simetrica", "asimetrica"),
                    size = 1, prob = c(0.1, 0.3, 0.6))
  if (destino == "apoptosis")  madres     <- madres     - 1
  if (destino == "simetrica")  madres     <- madres     + 1
  if (destino == "asimetrica") sanguineas <- sanguineas + 1
}

for (i in 1:sanguineas) {
  if (sample(c("muere", "vive"), 1, prob = c(0.2, 0.8)) == "muere")
    sanguineas <- sanguineas - 1
}

cat("Madres:", madres, "| Sanguíneas:", sanguineas, "\n")
salida

Paso 2

La versión rota

Metemos el código en una función para poder repetirlo. Parece correcto. No lo es.

simular_generacion <- function() {
  for (i in 1:madres) {
    destino <- sample(c("apoptosis","simetrica","asimetrica"),
                      1, prob = c(0.1, 0.3, 0.6))
    if (destino == "apoptosis")  madres     <- madres     - 1
    if (destino == "simetrica")  madres     <- madres     + 1
    if (destino == "asimetrica") sanguineas <- sanguineas + 1
  }
  for (i in 1:sanguineas) {
    if (sample(c("muere","vive"),1,prob=c(0.2,0.8))=="muere")
      sanguineas <- sanguineas - 1
  }
}

madres     <- 100
sanguineas <- 0

for (gen in 1:5) {
  simular_generacion()
  cat("Gen", gen, "→ Madres:", madres, "| Sanguíneas:", sanguineas, "\n")

¿Qué observas?

Los contadores siempre son 100 y 0. No cambian nunca. El código no da ningún error, pero tampoco hace nada. ¿Por qué?

¿Por qué madres y sanguineas no cambian fuera de la función?
APorque sample() no funciona dentro de funciones.
BPorque <- dentro de una función crea variables locales. Las globales no se tocan.
CPorque el bucle for no actualiza variables.
DPorque la función no tiene return().
Paso 3

Diagnóstico: ¿dónde viven las variables?

R tiene un sistema de entornos. Entender dónde vive cada variable es la clave para entender por qué el código falla.

Los dos entornos

Cuando R ejecuta simular_generacion(), crea un entorno local temporal. Todo lo que se asigna dentro de la función vive ahí — y desaparece cuando la función termina.

diagnostico.REjecutable
madres <- 100

ver_entorno <- function() {
  # R LEE madres del global (puede leer hacia fuera)
  cat("Dentro — madres vale:", madres, "\n")

  # <- crea una copia LOCAL, no modifica la global
  madres <- madres - 1
  cat("Dentro tras restar — madres vale:", madres, "\n")
}

ver_entorno()
cat("Fuera — madres sigue valiendo:", madres, "\n")
salida

Las tres reglas

Regla 1

Cada función crea su propio entorno local. Todo lo que se asigna con <- dentro vive ahí y desaparece al terminar.

Regla 2

R busca variables de dentro hacia fuera. Si no encuentra x en el entorno local, mira en el global. Pero nunca al revés.

Regla 3 — úsala con cuidado

<<- asigna en el entorno superior (global). Funciona, pero hace el código difícil de entender y depurar. Hay una solución más elegante.

reglas.REjecutable
# Regla 1: local desaparece
f1 <- function() { x <- 99 }
f1()
exists("x")  # FALSE — x no existe fuera

# Regla 2: R lee hacia fuera
y <- 42
f2 <- function() { cat("y vale", y, "\n") }
f2()  # encuentra y en el global

# Regla 3: <<- modifica el global (úsalo poco)
z <- 0
f3 <- function() { z <<- z + 1 }
f3(); f3()
cat("z vale", z, "\n")  # 2 — se modificó el global
salida
Paso 4

La solución elegante

Una función bien diseñada recibe todo lo que necesita por argumentos y devuelve todo lo que modifica. Eso la hace predecible, testeable y reutilizable.

Principio de función pura

Una función que no toca nada de fuera y devuelve todo lo que modifica se llama función pura. Es el estándar en bioinformática y en cualquier pipeline reproducible.

La función corregida

paso04_correcto.REjecutable
simular_generacion <- function(madres, sanguineas) {
  # ↑ recibe el estado por argumentos

  for (i in 1:madres) {
    destino <- sample(c("apoptosis","simetrica","asimetrica"),
                      1, prob = c(0.1, 0.3, 0.6))
    if (destino == "apoptosis")  madres     <- madres     - 1
    if (destino == "simetrica")  madres     <- madres     + 1
    if (destino == "asimetrica") sanguineas <- sanguineas + 1
  }
  for (i in 1:sanguineas) {
    if (sample(c("muere","vive"),1,prob=c(0.2,0.8))=="muere")
      sanguineas <- sanguineas - 1
  }

  return(list(madres = madres, sanguineas = sanguineas))
  # ↑ devuelve el nuevo estado
}

# Simulamos 10 generaciones
madres     <- 100
sanguineas <- 0

for (gen in 1:10) {
  resultado  <- simular_generacion(madres, sanguineas)
  madres     <- resultado$madres
  sanguineas <- resultado$sanguineas
  cat("Gen", gen, "→ Madres:", madres, "| Sanguíneas:", sanguineas, "\n")
}
salida

¿Qué cambió?

1
Argumentos explícitos

La función ya no lee variables del entorno global. Recibe madres y sanguineas como parámetros. Sus modificaciones son locales.

2
return(list(...))

La función devuelve el nuevo estado en una lista. El global solo se actualiza cuando tú decides hacerlo explícitamente.

3
Control explícito en el bucle

madres <- resultado$madres hace visible que estás actualizando el estado. No hay efectos ocultos.

El código que ejecuta el simulador

simulador_completo.REjecutable
simular_generacion <- function(madres, sanguineas,
                                  p_apo=0.1, p_sim=0.3, p_mue=0.2) {
  p_asi <- 1 - p_apo - p_sim
  for (i in 1:max(1, madres)) {
    d <- sample(c("a","s","x"), 1, prob=c(p_apo,p_sim,p_asi))
    if (d=="a" && madres>0) madres <- madres - 1
    if (d=="s") madres <- madres + 1
    if (d=="x") sanguineas <- sanguineas + 1
  }
  if (sanguineas > 0) {
    for (i in 1:sanguineas) {
      if (sample(c(0,1),1,prob=c(p_mue,1-p_mue))==0)
        sanguineas <- sanguineas - 1
    }
  }
  return(list(madres=madres, sanguineas=sanguineas))
}

# Parámetros — modifica estos valores
N0    <- 100   # población inicial
GENS  <- 10    # número de generaciones (recomendado: 5-25)
P_APO <- 0.10  # prob. apoptosis
P_SIM <- 0.30  # prob. división simétrica
P_MUE <- 0.20  # prob. muerte sanguínea

madres     <- N0
sanguineas <- 0
hist_m     <- c(N0)
hist_s     <- c(0)

for (gen in 1:GENS) {
  res        <- simular_generacion(madres, sanguineas, P_APO, P_SIM, P_MUE)
  madres     <- res$madres
  sanguineas <- res$sanguineas
  hist_m     <- c(hist_m, madres)
  hist_s     <- c(hist_s, sanguineas)
}

cat("Generación final — Madres:", madres, "| Sanguíneas:", sanguineas, "\n")
cat("\nHistorial por generación:\n")
for (i in 1:length(hist_m)) {
  cat(sprintf("  Gen %2d → Madres: %4d | Sanguíneas: %4d\n",
             i - 1, hist_m[i], hist_s[i]))
}
